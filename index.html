Instructor Efficiency Web Tool

This was my senior project at WWU. Me and a group of 3 other students worked with a professor to build a web tool that's goal was to build off of one of the professors previous projects. This previous project is called CAAML, and it uses machine learning to label parts of an audio file taken from a class room. These labels represent what is happening in the class room at that time. Some of the labels include lecture, instructor question, student question, group work, and silence. In total there are 9 possible labels. My team only needed a high level understanding of CAAML to build our web tool.

Our tool's purpose is to take those output labelling and create different graphs and visualizations that can convey the data in different ways. Our goal was to give instructors using the tool different insights into how they utilize class time and how they can better conduct class time. We also were tasked with creating a way to record and upload audio from the browser, as well as store these output files.

While weren't able to achieve all of the goals we set for ourselves, we did create 4 different visualizations, as well as learn a lot about the software development process. We used an agile scrum approach to our development, which included weekly meetings with the professor we were working with, as well as team meetings to stay on track.

If you'd like to hear more about this project, you can contact me and I can share some more information. As this project is still on going with a new team at WWU, I am unable to share the GitHub repo publicly.

Senior Project

Buy Better

This project was created for my Tech for Social Good class. The project's goal is to direct people who are shopping online on Amazon to order the same or similar product directly from the third part seller, or to buy a similar project in person at a local shop in their area. We wanted to have an easy way to learn about stores in our local area that could also have products we were shopping for on Amazon. We also wanted to show that it is possible to buy directly from a third party seller, rather than buy through Amazon.

To use the tool, a user would copy the URL from an Amazon product page, and paste it into the tool. They would also input their zip code, or use their exact location, whichever they prefer. Once they submit the form, if our tool determines that this product is being sold by a third party seller, a pop up will show, giving users the option to see if that seller has their own website so that they can buy directly from them. After closing the pop up, a map is shown with markers for each store that could have the same or similar product that the user is looking for. The stores are also listed beneath the map, as well as their address. There are also news articles listed to give more context for out tool, and to show how Amazon, and online shopping as a whole, has a negative impact on things like small and local businesses as well as the environment through packaging and shipping waste.

Our tool makes use of Google's Places API, that has data on stores that you can find on Google Maps. It searches for stores that are of the same category as the Amazon product, and also checks if the product is sold by a third party seller. As of now, the tool is not hosted for public use, but plans for that are in place.

Team Project

Movie Script Sentiment Analysis

This project was created when I took Natural Language Processing. I created the question "Can the highs and lows of movies be visualized using script sentiment analysis?" More specifically, can a movie be categorized as one of the six main arcs of storytelling. These six arcs originally stemmed from author Kurt Vonnegut's rejected masters thesis. The task I set out to accomplish was see if I could create an algorithm that read in a movie script, performed sentiment analysis on it, track and plot the sentiment, and see if that graph resembled any of the six arcs of storytelling.

I was successfully able to create an algorithm that did all those things that I set out to accomplish, and my results indicated that it is in fact possible to create graphs that resemble one of those six arcs. I was really pleased with my results, and even more happy that I was able to take two things that I love, movies and programming, and combine them to create something that I think is really cool and fascinating.

Microshell in C

This project was built using the C language, taking advantage of C system calls, as well as using C's pointer focused design and memory allocation for storing commands in arrays. It's main use is to read commands from the command line and appropriately execute it. The Microshell can handle many different commands from the terminal command line that most shells can handle.

This project helped me understand the UNIX operating system, gain experience writing C code, and helped me understand programming topics like pointers, system vs library calls, and memory. You can follow the github link to see a more in depth description of the commands possible in my Microshell.
